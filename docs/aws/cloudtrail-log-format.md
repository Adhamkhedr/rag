# CloudTrail Log Format

## Overview

Each CloudTrail log file contains one or more JSON records. Each record represents a single API call or event. Understanding the structure of these records is essential for security analysis, incident investigation, and automated log processing.

## Record Structure

A CloudTrail record contains the following top-level fields:

### eventVersion

The version of the log event format. The current version is `"1.09"`. Always check this field to ensure compatibility with your log parsing tools.

### userIdentity

A complex object that identifies the entity that made the request. It contains:

- **type**: The type of identity. Values include `Root`, `IAMUser`, `AssumedRole`, `FederatedUser`, `AWSService`, `AWSAccount`.
- **principalId**: A unique identifier for the entity. For IAM users, this is the user ID (e.g., `AIDAEXAMPLEUSERID`). For assumed roles, it is `<role-id>:<session-name>`.
- **arn**: The ARN of the principal that made the call (e.g., `arn:aws:iam::123456789012:user/alice`).
- **accountId**: The AWS account ID that owns the entity.
- **accessKeyId**: The access key used to sign the request. For temporary credentials, this begins with `ASIA`.
- **userName**: The friendly name of the identity (for IAM users).
- **sessionContext**: Present when temporary credentials are used. Contains `sessionIssuer` (details about the role or federated user) and `attributes` (creation time and MFA status).
- **invokedBy**: Present when the request was made by an AWS service on your behalf (e.g., `elasticmapreduce.amazonaws.com`).

### eventTime

The date and time the request was made, in UTC and ISO 8601 format (e.g., `"2024-01-15T14:32:08Z"`).

### eventSource

The AWS service that the request was made to (e.g., `iam.amazonaws.com`, `s3.amazonaws.com`, `ec2.amazonaws.com`, `sts.amazonaws.com`).

### eventName

The name of the API action that was called (e.g., `CreateUser`, `PutObject`, `RunInstances`, `AssumeRole`, `ConsoleLogin`).

### awsRegion

The AWS Region the request was made to (e.g., `us-east-1`, `eu-west-2`).

### sourceIPAddress

The IP address that the request was made from. For requests made by AWS services, this may be the service's DNS name (e.g., `ec2.amazonaws.com`). For console actions, this is the user's IP address.

### userAgent

The agent through which the request was made (e.g., `aws-cli/2.13.0`, `console.amazonaws.com`, `signin.amazonaws.com`, or a specific SDK agent string).

### requestParameters

A JSON object containing the parameters sent with the request. The contents vary by API action. For example, a `RunInstances` call includes `instanceType`, `imageId`, `subnetId`, etc. This field may be null for some events.

### responseElements

A JSON object containing the response from the service. For mutating (write) actions, this contains the results. For read-only actions (like `Describe*` or `Get*` calls), this is typically null. For example, a `CreateUser` response includes the new user's ARN and user ID.

### errorCode and errorMessage

Present only when the request resulted in an error. The `errorCode` contains the AWS error code (e.g., `AccessDenied`, `UnauthorizedAccess`, `EntityAlreadyExists`). The `errorMessage` provides a human-readable description.

### requestID and eventID

- **requestID**: The unique identifier for the API request, useful for correlating with AWS Support cases.
- **eventID**: A GUID generated by CloudTrail that uniquely identifies each event.

### readOnly

Boolean indicating whether the event is a read-only event (`true` for `Describe`, `Get`, `List` actions) or a write event (`false` for `Create`, `Delete`, `Put`, `Update` actions).

### resources

An array of resources involved in the event, each containing the resource ARN, account ID, and resource type.

### eventType

The type of event: `AwsApiCall`, `AwsConsoleSignIn`, `AwsServiceEvent`, or `AwsConsoleAction`.

## Example Records

### Console Sign-In Event

```json
{
  "eventVersion": "1.08",
  "userIdentity": {
    "type": "IAMUser",
    "principalId": "AIDAEXAMPLE123",
    "arn": "arn:aws:iam::123456789012:user/alice",
    "accountId": "123456789012",
    "userName": "alice"
  },
  "eventTime": "2024-01-15T09:15:23Z",
  "eventSource": "signin.amazonaws.com",
  "eventName": "ConsoleLogin",
  "awsRegion": "us-east-1",
  "sourceIPAddress": "203.0.113.50",
  "userAgent": "Mozilla/5.0",
  "requestParameters": null,
  "responseElements": {
    "ConsoleLogin": "Success"
  },
  "additionalEventData": {
    "MFAUsed": "Yes",
    "LoginTo": "https://console.aws.amazon.com/console/home"
  },
  "eventID": "a1b2c3d4-5678-90ab-cdef-EXAMPLE11111",
  "eventType": "AwsConsoleSignIn",
  "readOnly": false
}
```

### AssumeRole Event

```json
{
  "eventVersion": "1.08",
  "userIdentity": {
    "type": "IAMUser",
    "principalId": "AIDAEXAMPLE456",
    "arn": "arn:aws:iam::111122223333:user/bob",
    "accountId": "111122223333",
    "userName": "bob"
  },
  "eventTime": "2024-01-15T10:22:45Z",
  "eventSource": "sts.amazonaws.com",
  "eventName": "AssumeRole",
  "awsRegion": "us-east-1",
  "sourceIPAddress": "198.51.100.22",
  "userAgent": "aws-cli/2.13.0",
  "requestParameters": {
    "roleArn": "arn:aws:iam::123456789012:role/CrossAccountAdmin",
    "roleSessionName": "bob-session",
    "durationSeconds": 3600
  },
  "responseElements": {
    "credentials": {
      "accessKeyId": "ASIAEXAMPLEKEY",
      "expiration": "Jan 15, 2024 11:22:45 AM",
      "sessionToken": "FwoGZXIvY..."
    },
    "assumedRoleUser": {
      "assumedRoleId": "AROAEXAMPLEROLE:bob-session",
      "arn": "arn:aws:sts::123456789012:assumed-role/CrossAccountAdmin/bob-session"
    }
  },
  "eventID": "a1b2c3d4-5678-90ab-cdef-EXAMPLE22222",
  "eventType": "AwsApiCall",
  "readOnly": false
}
```

### Access Denied Example

```json
{
  "eventVersion": "1.08",
  "userIdentity": {
    "type": "AssumedRole",
    "principalId": "AROAEXAMPLE789:lambda-session",
    "arn": "arn:aws:sts::123456789012:assumed-role/LambdaRole/lambda-session",
    "accountId": "123456789012"
  },
  "eventTime": "2024-01-15T12:05:33Z",
  "eventSource": "s3.amazonaws.com",
  "eventName": "GetObject",
  "awsRegion": "us-east-1",
  "sourceIPAddress": "10.0.1.50",
  "userAgent": "aws-sdk-python/1.28.0",
  "errorCode": "AccessDenied",
  "errorMessage": "Access Denied",
  "requestParameters": {
    "bucketName": "confidential-data-bucket",
    "key": "secrets/credentials.json"
  },
  "responseElements": null,
  "eventID": "a1b2c3d4-5678-90ab-cdef-EXAMPLE33333",
  "eventType": "AwsApiCall",
  "readOnly": true
}
```
